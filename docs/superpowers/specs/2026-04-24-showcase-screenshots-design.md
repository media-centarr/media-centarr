# Showcase Screenshot Expansion — Design

**Date:** 2026-04-24
**Status:** Approved for implementation planning

## Problem

The showcase instance exists to produce screenshots for marketing (`docs-site/index.html`) and user documentation (GitHub wiki). Current coverage is thin:

- Only 4 screenshots exist: `library-grid`, `library-detail-movie`, `library-detail-tv`, `settings-overview`.
- Two wiki pages reference screenshots the tour never generates: `Review-Queue.md` → `review-queue.png` and `Release-Tracking.md` → `release-tracking.png` (broken image links in production wiki).
- Four full LiveView surfaces are never captured: `/review`, `/history`, `/download`, `/console`.
- The TV detail shot (Pioneer One) falls back to loud gradient thumbnails because TMDB has no `still_path` for indie/CC TV series.
- Ten wiki pages have no imagery despite describing visual UI: `First-Run`, `Adding-Your-Library`, `Playback`, `Keyboard-and-Gamepad`, `Download-Clients`, `Prowlarr-Integration`, `Watch-History`, `TMDB-API-Key`, etc.

The result: the wiki has broken links today, the landing page is marketing-thin, and the TV detail looks unfinished.

## Scope

Target: approximately 17 screenshots covering every user-facing surface and landing every wiki page that describes visual UI. Includes one catalog change (swap Dragnet for Beverly Hillbillies S1), a redesigned fallback thumbnail fixture, seed-data extensions for populated history + console logs, and one belt-and-suspenders safety assertion against direct IEx seeder invocation.

Out of scope:
- First-run / empty-library screenshots (would require a separate empty DB binding — double the infrastructure for one shot)
- Live playback overlay screenshots (mpv overlay can't be captured from headless Playwright)
- A/B variants (light/dark, sidebar-expanded) — dark+collapsed only, to keep the tour maintainable

## Design

### 1. Catalog changes

Edit `lib/media_centarr/showcase/catalog.ex`:

- **Add** Beverly Hillbillies S1 (1962) — all 36 S1 episodes are US public domain (copyright-renewal lapse), and TMDB has both series metadata and episodic `still_path` coverage.
- **Keep** Pioneer One S1 — CC-BY-NC-SA, retains the "indie/CC series" narrative in the catalog and exercises the fallback-thumbnail code path.
- **Drop** Dragnet 1951 — TMDB coverage thin, Beverly Hillbillies replaces its role.

Legal footing: the showcase already uses TMDB posters for copyrighted-but-PD-status films (Nosferatu, Metropolis, etc.). TMDB's own position is that image copyright stays with original rights holders, not TMDB. Extending this stance from PD films to PD-status TV episodes is consistent, not a new leap. Beverly Hillbillies theme song is still copyrighted; irrelevant since we don't use audio.

### 2. Fallback thumbnail redesign

Replace the 5 gradient PNGs at `priv/showcase/fixtures/thumbs/thumb-{1..5}.jpg` with tasteful dark-theme placeholders:

- Dark neutral background matching the app's ink palette (`#0a0a0f`-ish).
- Single subtle monochrome icon centered — play triangle / film reel / TV glyph / etc., one per fixture for variety.
- Very soft noise/grain so the surface doesn't read as "rendering bug flat rectangle."
- 400×225 (TMDB's minimum episodic-backdrop size, so the aspect ratio survives the detail modal's thumb grid).

Regenerated by `scripts/generate-showcase-thumbs` (script already exists; content generator will be rewritten). Used only on code path `MediaCentarr.Showcase.bundle_episode_thumb!/1`, which now only fires for Pioneer One.

### 3. Seed scenario additions

Extend `MediaCentarr.Showcase.seed!/0`:

- **Populated watch history:** 15–20 events across last 30 days, mixing movies and episodes. Current seeder creates 5 — enough to show the shape but not enough to feel like a real history.
- **Console log entries:** emit a varied `Log.info/warning/error` mix across components during seeding so `/console` shows content when captured. Existing seed already logs failures; audit coverage and fill gaps for `:pipeline`, `:tmdb`, `:library`, `:watcher`.
- **Acquisition scenarios:** 2–3 items across pending / in-progress / completed states, if the `acquisition_` schema supports static seed data. If the page requires a live job running to render, skip the `/download` stop and document the gap in the tour file comments.
- **Keyboard focus shot:** handled in tour JS (programmatic focus before screenshot), not seed data.
- **Release tracking:** seeder already creates upcoming + available-now; verify the `/status` release card renders them without further changes.

### 6. Safety

No new defense infrastructure — existing guards (`MEDIA_CENTARR_CONFIG_OVERRIDE` required for `mix seed.showcase`, showcase TOML pins its own DB and watch_dirs under `priv/showcase/`) still cover every new code path since all new seed logic goes through existing `Library.*`, `ReleaseTracking.*`, `WatchHistory.*` facades.

**One additive belt-and-suspenders check:** `MediaCentarr.Showcase.seed!/0` asserts at entry that `MediaCentarr.Config.get(:database_path)` contains the literal substring `"showcase"`. Raises if not. Today the Mix task wrapper is the only guard; direct IEx invocation of `MediaCentarr.Showcase.seed!/0` bypasses the env-var check. This assertion is the last-ditch safety rail.

## Execution order

1. **Catalog + fallback thumbnails.** Catalog swap (`Catalog.tv_series/0`), fallback fixture regeneration, verify with a reseed.
2. **Seed scenarios.** Populated watch history, console log seeding, acquisition seed if feasible. One pass of `mix seed.showcase` to confirm everything lands.
3. **Tour rewrite.** Add 13 new stops to `screenshot.tour.js`, add scroll-target anchors to settings HTML where missing.
4. **Iteration.** Run tour, review each shot, adjust selectors/waits/scroll targets until each looks right. This loop is the bulk of the work.
5. **Landing page + wiki integration.** Update `docs-site/index.html`, batch wiki commits.
6. **Ship.** `/ship patch` for the backend repo (catalog + seeder + tour changes); separate `jj describe + push` for the wiki repo.

## Known gaps accepted

- No first-run / empty-state screenshot.
- No playback overlay screenshot.
- No light-theme variants.
- No sidebar-expanded variants.

If any of these become important later, each is a separate small design, not a scope expansion of this one.
